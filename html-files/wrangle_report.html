Data Assessment

After I downloaded these 3 files (DataFrames), I was ready to check for data quality (content) and tidiness (structural) issues.

  • Quality issues - examples range from duplicated or missing values i.e. data entry that should not be there in the first place.
  • Tidiness issues - structural problems generally prevent easy analysis. Examples include multiple observations in each row, non-variable column names, various observational units in a single table etc.

Udacity provided general principles to look for especially in data quality assessment, involving:

  1. Completeness - detecting if there are missing records, rows, columns or cells.
  2. Validity - data not defined to a schema i.e. having negative values for height/weight.
  3. Accuracy - the data entry is seemingly correct but has a faulty from the start i.e. incorrect name spellings
  4. Consistency - data is ostensibly valid and accurate, but not operated in a cohesive manner in a standard format across/within tables, such as the acronym ‘NY’  and ‘New York’ in various rows, when only one of these should be used throughout.
